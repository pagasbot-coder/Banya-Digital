# Обзор для руководства — Banya-Digital ERP

Понятное описание проекта без технического жаргона. Для владельца, операционного и финансового менеджмента.

---

## Что это за система

**Banya-Digital ERP** — единая цифровая система для **премиального банно-спа комплекса** (бренд в интерфейсе: «Дегтярные Бани · ERP»).

Сейчас это не «ещё одна таблица в Excel», а задел под то, чтобы в одном месте видеть:

- сколько зарабатывают залы и насколько они загружены;
- какая маржа за день;
- что требует внимания на смене (склад, кухня, простои);
- как идут чеклисты открытия/закрытия смены.

Целевая аудитория системы: **владелец**, **финансы**, **операционный менеджер**, **склад**, **администратор броней**.

---

## Модули простыми словами

| Раздел в меню | Для кого | Что даёт |
|---------------|----------|----------|
| **Сводка** | Владелец, ops | Утренняя картина дня: выручка, загрузка залов, маржа, важные тревоги, ход смены |
| **Финансы** | Владелец, финдиректор | Unit economics за сегодня: выручка, COGS, маржа по каждому залу |
| **CRM** | Администратор | Список гостей и все брони на сегодня (время, зал, статус) |
| **Операции** | Ops, склад | Расписание spa-программ, слоты кухни (в т.ч. конфликты SLA), переход к чеклистам на сводке |

Все четыре раздела меню работают на **демо-данных PostgreSQL** после `npm run db:seed`.

---

## Что уже сделано

- Каркас системы: четыре модуля в меню, единый премиальный интерфейс (тёмная тема, фирменные шрифты).
- **Сводка на русском языке:**
  - загрузка залов и выручка за день / неделю / месяц;
  - маржа в процентах;
  - список критических ситуаций (склад, конфликты расписания, простои);
  - блок операций смены: чеклисты с прогрессом «выполнено N из M» и отметками галочками;
  - сравнение с предыдущим периодом (на демо-данных).
- **Демо-данные в базе** для показа и обучения: четыре зала, гости, брони, склад органики, финансовые строки за сегодня и вчера — без ручного ввода при первом запуске.
- **Финансы** — таблица unit economics по залам за текущий день.
- **CRM** — гости и бронирования на сегодня.
- **Операции** — программы spa, слоты кухни (конфликты подсвечены), ссылка на чеклисты смены.
- Инструкции деплоя: GitHub + Vercel (`docs/GITHUB-DEPLOY.md`).
- Документ с требованиями к продукту и измеримыми целями пилота (`knowledge-base/product-brief.md`).

## Как измерим успех пилота (8 недель)

Цели из продуктовой спецификации — без IT-терминов:

| Цель | Было | Хотим к концу пилота |
|------|------|----------------------|
| Время закрытия смены | ~45 минут в таблицах | **На 30% быстрее** (около 31 минуты) |
| Точность остатков сена и пихты | Примерно 70% «на глаз» | **Не ниже 95%** при сверке со складом |
| Видимость маржи по залам | Раз в неделю вручную | **Каждый день** на сводке |
| Конфликты «программа ↔ кухня» | Не отслеживались | **Ноль необработанных** за смену |
| Доверие к цифрам на сводке | Учебные заглушки | **Все ключевые показатели из одной базы** |

## Как посмотреть демо

На компьютере с установленным проектом (см. README для команды):

1. Запустить базу данных (Docker).
2. Загрузить демо-данные одной командой.
3. Открыть в браузере: **Сводка**, **Финансы**, **CRM**, **Операции**.

Без базы система покажет подсказку, что нужно подключить данные — не «сломается» молча.
