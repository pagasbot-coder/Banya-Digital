# Регламент пилота Banya-Digital ERP (8 недель)

**Версия:** 1.0 · **Дата:** 2026-05-26  
**Аудитория:** владелец, операционный менеджер, старший администратор, склад (один комплекс)  
**Связанные документы:** `@knowledge-base/product-brief.md`, `@docs/management-overview.md`

> Демо-система **не является** бухгалтерской или налоговой отчётностью. Цифры пилота — для операционных решений и согласованных метрик, не для сдачи в ФНС.

---

## 1. Цели пилота

| Цель | Метрика | Ориентир |
|------|---------|----------|
| **North Star (primary)** | **WAMZ** — Weekly Active Managed Zones | К **неделе 8:** данные по **≥3 из 4** залов за каждую скользящую 7-дневку (выручка/COGS, чеклист или бронь — см. `product-brief.md`) |
| **Co-metric** | Время закрытия смены | С **45 мин** до **≤30 мин** за счёт чеклиста и ввода в систему |
| **Co-metric** | Видимость маржи по залам | Dashboard с daily margin — поддерживает решения владельца, не заменяет WAMZ |
| Качество данных | Точность остатков органики | **≥95%** после еженедельной сверки склада |
| Операционный контроль | Чеклисты смены | **N/M = 100%** в дни с полной сменой (кроме форс-мажора) |

**Срок:** 8 календарных недель с даты «Старт пилота» (фиксирует PM с владельцем).

**North Star WAMZ (кратко):** успех пилота — не «открыл dashboard утром», а **регулярный ввод данных по зонам**. Каждую неделю PM считает, сколько из **четырёх залов** объекта имели хотя бы одно действие в системе за последние 7 дней: проводка выручки или COGS, отметка в чеклисте смены или обновление брони. Цель к финалу — **три зоны из четырёх** стабильно «живые»; идеал — все четыре.

## 2. Участники и роли

| Роль | Кто на объекте | Ответственность в пилоте |
|------|----------------|---------------------------|
| **Владелец / финдиректор** | Собственник или CFO | Утренняя сводка, решения по марже и yield |
| **Операционный менеджер** | Управляющий сменой | Чеклисты, конфликты кухня↔SPA, закрытие смены |
| **Старший администратор** | Админ зала / CRM | Брони и гости (после T-010); дисциплина «бронь = факт» |
| **Склад / закупки** | Кладовщик или ответственный за органику | FIFO OUT перед закрытием, еженедельная сверка остатков |
| **Human Architect + PM** | Команда продукта | Регламент, метрики, эскалации, итог пилота |

**Доступ к системе в пилоте:** один общий вход в браузере (до внедрения T-009 Auth). Не передавать URL посторонним; не использовать реальные телефоны гостей в скриншотах.

## 3. Ритм работы

| Частота | Действие | Где в системе |
|---------|----------|---------------|
| **Ежедневно, утро** | Сводка: KPI, алерты, чеклисты | `/dashboard` |
| **Ежедневно, в течение смены** | Ввод выручки и COGS при необходимости | `/finance` |
| **Ежедневно, перед закрытием** | Закрыть чеклист N/M; FIFO OUT органики при списании | `/dashboard`, `/operations/inventory` |
| **Еженедельно** | Сверка остатков сена/пихты с фактом на складе | Склад + PM (протокол в 1 страницу) |
| **Еженедельно** | 15-мин созвон: что мешало, что улучшить | PM / владелец |
| **Недели 1, 4, 8** | UAT по чеклисту QA (`qa-checklist.md` § Pilot) | QA |

**Бизнес-день:** календарный день по **Europe/Moscow** (как в seed и отчётах).

## 4. Критерии успеха и остановки

### Успех пилота (все пункты к концу 8-й недели)

1. **North Star (WAMZ):** на неделях **6–8** среднее **WAMZ ≥ 3/4** по скользящей 7-дневке (PM сверяет с `product-brief.md`; stretch — **4/4** две недели подряд).
2. Co-metric: среднее время закрытия смены **≤30 мин** (опрос + выборочное наблюдение).
3. Остатки органики: расхождение с фактом **≤5%** по итогам еженедельной сверки.
4. Чеклисты: **≥90%** смен с N/M = 100% или объяснённым исключением.
5. Нет критических инцидентов потери данных (backup по `devops-runbook` при production).

### Остановка досрочно (stop criteria)

- Владелец отказывается от ежедневного ритма **2 недели подряд** без согласованной замены метрики.
- Система недоступна **>24 ч** на production без плана восстановления.
- Выявлено несоответствие 152-ФЗ по хранению ПДн — до решения legal/Architect.

## 6. Что не входит в пилот

- Мультифилиал и отдельные tenant
- Интеграция YCLIENTS / 1С (T-023, T-024 — BLOCKED)
- Зарплаты мастеров, полная бухгалтерия, НДС
- Маркетинг B2C performance и публичная запись гостя
