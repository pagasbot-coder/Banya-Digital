# Product Brief — Banya-Digital ERP

**Премиальный ERP/CRM для банно-спа комплекса:** unit economics, загрузка залов, синхронизация программ и кухни, FIFO-склад органики.

## Проблема

Операции премиальной бани разрознены: финансы в таблицах, загрузка залов «на глаз», тайминги мастеров и кухни не синхронизированы, учёт сена/пихты без FIFO. Нет единой картины маржинальности и yield.

## Аудитория

- Владелец / финансовый директор — unit economics, P&L по залам и услугам
- Операционный менеджер — динамическая загрузка, чеклисты, тайминги
- Склад / закупки — органический FIFO (сено, пихта)
- Администраторы и CRM — гости, брони, программы

## MVP (v1)

| Модуль | Фокус |
|--------|--------|
| **finance** | Unit economics: выручка, COGS, маржа по залу/услуге/дню |
| **operations** | Загрузка залов (yield), чеклисты, тайминги spa + kitchen sync |
| **operations/inventory** | FIFO склад органики (hay, fir) |
| **crm** | Гости, брони, программы (базовый контур) |
| **dashboard** | KPI: загрузка, маржа, критические алерты |

## Вне scope (не делаем в v1)

- Полная бухгалтерия / 1С-интеграция
- Мобильное приложение для гостей
- AI-прогнозирование спроса (после накопления данных)

## Метрики успеха

- Время закрытия смены: −30% vs таблицы
- Точность остатков органики: ≥95% (FIFO)
- Видимость маржи по залу: daily, без ручной сводки
- Синхронизация kitchen ↔ spa program: 0 конфликтов слотов на пилоте
